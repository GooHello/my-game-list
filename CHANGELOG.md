### 🚨 防幻觉 Skill 备忘录 (Anti-Hallucination Skill)

**教训 1：永远不要相信“智能识别”的完备性**
- **幻觉表现**：我以为写几个关键词（“手游”、“移动版”）和一份几十游戏的名单，就能“智能”识别出所有的手游。结果漏掉了《非人学园》等大量游戏。
- **Skill 准则**：对于分类（如手游、GAAS），**必须依赖外部权威数据源（如 TapTap 标签、IGDB 平台数据）**，或者**要求用户提供一份绝对完整的名单**。绝不能用 AI 的“常识”去硬猜，因为 AI 的常识是有盲区的。

**教训 2：永远不要在前端写死业务逻辑**
- **幻觉表现**：为了图快，我在 `page.tsx` 里写死了 `anchorList` 和 `mobileList`，甚至写死了 `filter(game => !game.title.includes('光环无限（战役）'))`。这导致底层数据和前端展示严重脱节，图片路径匹配失败。
- **Skill 准则**：**前端只负责渲染，业务逻辑必须下沉到数据层（Excel/JSON）**。要删除游戏，就从 Excel 里删；要分类，就在 Excel 里打 Tag。

**教训 3：永远不要在没有验证数据源的情况下执行批量修改**
- **幻觉表现**：我用一个错误的备份文件覆盖了最新的数据，导致《七日世界》等新加的游戏丢失，图片路径错乱。
- **Skill 准则**：在执行任何修改脚本前，**必须先打印出目标数据的前几行进行人工确认**。

**教训 4：永远保证开发服务器处于活动状态 (Always-On Dev Server)**
- **幻觉表现**：在执行完数据修改脚本后，我经常忘记重启或检查 Next.js 开发服务器，导致用户刷新页面时看到“拒绝连接”，严重打断了用户的 Review 心流。
- **Skill 准则**：**每次执行完任何修改数据的脚本后，必须强制检查并启动 `npm run dev`，确保用户能第一时间看到修改结果。**

**教训 5：永远不要在数据转换时“想当然”地拼接路径 (Path Assumption)**
- **幻觉表现**：在 `import-excel.js` 中，我为了图省事，直接把所有游戏的封面路径写死为 `/covers/${id}.jpg`。这导致那些从 Bing 或 TapTap 抓取并带有前缀（如 `bing_xxx.jpg`）的图片全部 404 消失。
- **Skill 准则**：**在生成文件路径时，必须去真实的文件系统中进行校验（`fs.existsSync` 或 `fs.readdirSync`）**。如果文件存在多种命名可能，必须遍历匹配真实存在的文件名，绝不能靠“猜”或“写死”。
